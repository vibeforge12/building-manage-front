# 초기 네비게이션 및 화면 계획

## 개요

백엔드가 없는 상태에서 역할별 로그인 진입 흐름을 구현하기 위해 필요한 화면과 전환 계획을 정리한다. 기본 홈 화면에서 시작하여 사용자/관리자 로그인 선택, 세부 로그인 폼 진입까지를 다룬다.

## 화면 구성

1. **메인 홈 화면**
   - 상태: 작업완료
   - 버튼: `유저 로그인`, `관리자 로그인`, `회원가입`.
   - 백그라운드: `assets/home.png` 전체 화면 적용.
   - 역할: 사용자에게 초기 선택지를 제공하고 각 플로우로 이동.

2. **유저 로그인 폼 화면**
   - 상태: 작업완료
   - 입력: `동`, `호수`, `비밀번호`.
   - 행동: 로그인 제출, 비밀번호 찾기 링크, 회원가입 이동 버튼.
   - 백그라운드: 단색(백그라운드 이미지는 적용하지 않음).

3. **관리자 로그인 선택 화면**
   - 버튼: `본사 로그인`, `관리자/담당자 로그인`.
   - 설명 텍스트로 두 타입의 차이를 안내.
   - 백그라운드: `assets/home.png` 전체 화면 적용.

4. **본사 로그인 폼 화면**
   - 상태: 작업완료
   - 입력: `이메일`, `비밀번호`.
   - 디자인: 유저 로그인 폼과 동일 레이아웃 사용하되 `동`/`호수` 입력을 `이메일` 입력으로 대체.
   - 행동: 로그인 버튼만 제공(회원가입/비밀번호 찾기 링크 없음). 실패 시 `이메일 또는 비밀번호가 잘못 되었습니다. 정확한 정보를 입력해 주세요.` 안내 문구 노출. 본사 계정은 내부에서 직접 발급.
   - 백그라운드: 단색(이미지 미적용).

5. **관리자/담당자 로그인 폼 화면**
   - 상태: 작업완료
   - 입력: 관리자 코드 단일 필드.
   - 행동: 로그인 버튼. 코드 검증 실패 시 `관리자 코드가 잘못 되었습니다. 관리자 코드를 정확히 입력해 주세요.` 안내 문구 출력.
   - 백그라운드: 단색(이미지 미적용).

6. **회원가입 화면**
   - 입력: 기본 사용자 정보 및 역할 선택(임시로 일반 유저만 허용 가능).
   - 백그라운드: `assets/home.png` 전체 화면 적용.

## 네비게이션 흐름

- `MainHomeScreen`
  - `유저 로그인` → `UserLoginScreen`
  - `관리자 로그인` → `AdminLoginSelectionScreen`
  - `회원가입` → `SignUpScreen`
- `AdminLoginSelectionScreen`
  - `본사 로그인` → `HeadquartersLoginScreen`
  - `관리자/담당자 로그인` → `ManagerStaffLoginScreen`
- 각 로그인 제출 성공 시점에는 임시로 역할별 대시보드 화면으로 연결할 수 있는 placeholder 라우트를 준비.

## UI/테마 고려 사항

- `assets/home.png`를 전체 배경으로 사용하는 화면은 `Stack` 또는 `DecoratedBox`를 사용해 전체 크기에 맞춰 렌더링한다.
- 폼 화면에서는 가독성을 위해 기본 테마 색상 또는 밝은 배경을 유지한다.
- 버튼과 입력 필드는 Material 3 스타일을 따라 일관성 있게 구성한다.
- 모든 화면의 헤더 텍스트는 `PageHeaderText` 위젯으로 통일하고, 색상(`#464A4D`), 폰트 크기(16), 두께(FontWeight.w700)를 유지한다.

## 구현 순서 제안

1. `assets/home.png`를 프로젝트에 추가하고 `pubspec.yaml`에 등록.
2. 라우팅 구조(예: `go_router` 또는 `Navigator 2.0`) 골격을 만들고 각 화면을 스텁 위젯으로 구성.
3. 메인 홈 → 선택 화면 → 로그인 폼 화면 간 네비게이션을 연결.
4. 각 화면에서 백그라운드 이미지 적용 여부를 반영하고 UI 스케폴딩.
5. 로그인 폼 제출 시 임시로 콘솔 출력 또는 스낵바로 결과를 확인하도록 처리.

## 향후 고려 사항

- 실제 백엔드 API 연동 시 로그인 폼 검증 및 상태관리 로직 추가.
- 역할 선택에 따라 회원가입 폼 항목 차별화.
- 관리자/담당자와 본사의 권한 차이를 반영한 역할 기반 대시보드 연결.
