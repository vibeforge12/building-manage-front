# 건물관리 플랫폼 개발 계획

## 프로젝트 개요
하나의 Flutter 앱에서 4가지 유저 타입을 지원하는 건물관리 시스템

### 유저 타입
1. **유저 (일반 거주자)**
2. **관리자 (건물 관리자)**
3. **담당자 (유지보수 담당자)**
4. **본사 (관리 본사)**

## 아키텍처 설계

### 1. 인증 시스템
- 통합 로그인 화면
- 유저 타입별 권한 관리
- JWT 토큰 기반 인증
- 자동 로그인 및 세션 관리

### 2. 라우팅 구조
```
/login → 통합 로그인
├── /user → 일반 유저 대시보드
├── /admin → 관리자 대시보드
├── /manager → 담당자 대시보드
└── /headquarters → 본사 대시보드
```

### 3. 상태 관리
- Provider 또는 Riverpod 사용
- 유저 정보 및 권한 상태 전역 관리
- 각 유저 타입별 상태 분리

## 기술 스택

### Frontend (Flutter)
- **상태 관리**: Riverpod
- **라우팅**: go_router
- **HTTP 클라이언트**: dio
- **로컬 저장소**: shared_preferences, hive
- **UI 컴포넌트**: Material 3

### 폴더 구조
```
lib/
├── core/           # 핵심 유틸리티
├── data/           # 데이터 레이어
├── domain/         # 비즈니스 로직
├── presentation/   # UI 레이어
│   ├── auth/       # 인증 관련
│   ├── user/       # 유저 화면
│   ├── admin/      # 관리자 화면
│   ├── manager/    # 담당자 화면
│   ├── headquarters/ # 본사 화면
│   └── common/     # 공통 컴포넌트
└── main.dart
```
